
# 3.3 REST API Integration Scenario

The company wants to automatically push **daily sales reports** to their business intelligence platform via a REST API.

API Endpoint:

POST https://api.bi-platform.com/v1/reports

Authentication:

Bearer token in the `Authorization` header.

---

## API Payload Format

```json
{
  "report_type": "daily_sales",
  "date": "2024-11-29",
  "data": {
    "total_revenue": 125000.50,
    "total_orders": 450,
    "average_order_value": 277.78,
    "top_category": "Electronics"
  },
  "generated_at": "2024-11-29T08:00:00Z"
}
```

---

# Implementation Approach

The existing Go automation tool will be extended to support generating a **daily sales summary** and sending the result to the BI platform API.

The workflow is:

1. Query daily sales data from PostgreSQL.
2. Format the data into the required JSON structure.
3. Send the JSON payload to the API endpoint.
4. Handle API errors and implement retries.

---

# Go Implementation Example

Below is a simplified implementation (`api_integration.go`).

```go
package api

import (
    "bytes"
    "context"
    "database/sql"
    "encoding/json"
    "fmt"
    "log"
    "net/http"
    "os"
    "time"
)

type DailySalesReport struct {
    ReportType  string      `json:"report_type"`
    Date        string      `json:"date"`
    Data        ReportData  `json:"data"`
    GeneratedAt string      `json:"generated_at"`
}

type ReportData struct {
    TotalRevenue      float64 `json:"total_revenue"`
    TotalOrders       int     `json:"total_orders"`
    AverageOrderValue float64 `json:"average_order_value"`
    TopCategory       string  `json:"top_category"`
}

const apiURL = "https://api.bi-platform.com/v1/reports"

func GenerateDailySales(db *sql.DB, reportDate string) (*DailySalesReport, error) {

    query := `
    SELECT
        SUM(total_amount),
        COUNT(id),
        AVG(total_amount)
    FROM orders
    WHERE status='completed'
    AND DATE(order_date)=$1
    `

    var revenue float64
    var orders int
    var avg float64

    err := db.QueryRowContext(context.Background(), query, reportDate).
        Scan(&revenue, &orders, &avg)

    if err != nil {
        return nil, err
    }

    report := &DailySalesReport{
        ReportType: "daily_sales",
        Date: reportDate,
        Data: ReportData{
            TotalRevenue: revenue,
            TotalOrders: orders,
            AverageOrderValue: avg,
            TopCategory: "unknown",
        },
        GeneratedAt: time.Now().UTC().Format(time.RFC3339),
    }

    return report, nil
}

func SendReport(report *DailySalesReport) error {

    token := os.Getenv("BI_API_TOKEN")

    payload, err := json.Marshal(report)
    if err != nil {
        return err
    }

    req, err := http.NewRequest("POST", apiURL, bytes.NewBuffer(payload))
    if err != nil {
        return err
    }

    req.Header.Set("Authorization", "Bearer "+token)
    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{Timeout: 10 * time.Second}

    var resp *http.Response

    for i := 0; i < 3; i++ {

        resp, err = client.Do(req)

        if err == nil && resp.StatusCode < 300 {
            log.Println("Report sent successfully")
            return nil
        }

        log.Println("Retry attempt", i+1)
        time.Sleep(2 * time.Second)
    }

    return fmt.Errorf("failed to send report after retries")
}
```

# Scheduling the Automation

This report can be scheduled using a **cron job**.

Example cron configuration:

```
0 8 * * * go run cmd/main.go --report=daily-sales-api
```

This will execute the report generation and API push **every day at 08:00 AM**.

---

# Error Handling Strategy

The system handles failures using:

- API retry mechanism (3 attempts)
- Logging errors to application logs
- Timeout protection for HTTP requests

These measures ensure reliability when communicating with the external BI platform.
